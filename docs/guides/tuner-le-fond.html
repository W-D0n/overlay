Fond seul dans OBS et page de tuning

Ce guide couvre le mode background-only : une URL OBS qui ne rend que l'animation de fond, réglée en direct depuis la page de tuning. Le moteur de scènes complet (index.html) n'est pas concerné ici.

Vue d'ensemble

Quatre pièces communiquent :

  • background.html — la page à mettre dans OBS. Elle ne fait qu'une chose : afficher l'effet de fond courant.
  • dev/studio.html — l'entrée de création commune aux fonds et aux scènes.
  • dev/background-tuner.html — la page de tuning. Aperçu plein écran + barre latérale rétractable avec la liste des effets, leurs paramètres et les presets.
  • Le serveur d'état (port 4462, lancé automatiquement) — retient le dernier réglage dans dev/data/background-state.json et pousse chaque changement en direct vers OBS.

Concrètement : tu bouges un curseur dans le tuner → l'aperçu change immédiatement → OBS suit dans la seconde → et au prochain démarrage, OBS recharge le dernier état sauvegardé tout seul.

Mettre le fond dans OBS

  1. Lancer start-stream.bat (ou start-dev.bat pour bricoler).
  2. Dans OBS : Source → Navigateur (Browser Source).
  3. URL : http://localhost:5500/background.html
  4. Largeur 1920, hauteur 1080.

Laisser le champ CSS personnalisé vide. OBS injecte ce CSS après celui de la page : une règle ajoutée ici peut forcer un fond, masquer un élément ou changer une taille, mais crée alors une différence entre le tuner et OBS. Pour superposer l'animation sur une capture, utiliser http://localhost:5500/background.html?transparent=1 plutôt qu'une surcharge CSS.

Si le serveur n'est pas encore lancé quand OBS charge la page, ce n'est pas grave : la page attend et se reconnecte toute seule (1 s → 30 s max entre les tentatives).

La page de tuning

Ouvrir http://localhost:5500/dev/studio.html (s'ouvre automatiquement avec start-dev.bat), puis choisir Fonds & presets. L'URL directe dev/background-tuner.html reste disponible pour le diagnostic.

  • La barre latérale se rétracte avec le bouton ◀ / ▶ sur son bord — pratique pour juger le rendu plein écran.
  • Effet : le menu déroulant liste toutes les animations disponibles (+ « (aucun) » pour couper le fond).
  • Paramètres : chaque nombre dispose d'un curseur borné, d'une valeur précise et de son unité. Chaque changement est appliqué en direct.
  • Bibliothèque Atelier : appliquer teste une ambiance ; Ajouter en crée une copie personnelle.
  • Recherche : le champ commun filtre les deux bibliothèques par nom, effet ou tag, sans dépendre des accents ni des majuscules.
  • Mes presets : Créer enregistre le réglage courant. URL copie l'adresse stable ; ✎ renomme sans la casser ; ⧉ duplique ; ✕ supprime avec confirmation. Le champ inférieur accepte des tags séparés par des virgules. Nouveau quitte le mode de mise à jour du preset chargé.
  • Sauvegarde portable : Exporter télécharge la bibliothèque personnelle dans un JSON versionné. Importer valide tout le fichier avant écriture, conserve les identifiants connus et suffixe les conflits de nom sans écraser un autre preset. Le tuner affiche d'abord combien de presets seront créés, mis à jour ou renommés, puis attend Confirmer l'import. Annuler ou sélectionner un fichier invalide ne modifie rien. Si un autre onglet change la bibliothèque entre-temps, le résumé est recalculé et doit être confirmé à nouveau.
  • Qualité de rendu : Auto plafonne le DPR à 2 ; Performance OBS le limite à 1. La ligne voisine affiche FPS et DPR.

La ligne de statut en bas de la barre indique si le serveur d'état répond. Si elle est rouge, lancer start-dev.bat ou start-stream.bat.

Couleurs, palettes et scènes

  • tokens.css contient les couleurs de l'identité Atelier utilisées par l'interface.
  • Les couleurs des effets acceptent une valeur CSS libre : hex, rgb(...), oklch(...) ou var(--nom-du-token).
  • components/color-palette.json contient les couleurs nommées et les gradients de travail destinés au tuner.

Deux modes sont disponibles :

  • background.html suit l'état courant du tuner ; toutes les sources utilisant cette URL affichent le même effet.
  • background.html?preset=identifiant-stable&transparent=1 reste attachée à ce preset, même après renommage. Les anciennes URL par nom restent compatibles.

Pour avoir un fond différent par scène, créer les presets, utiliser leur bouton URL, puis ajouter une source Navigateur par URL dans OBS. Activer Arrêter la source lorsqu'elle n'est pas visible pour ne laisser tourner que la scène affichée.

Ajouter une nouvelle animation — sans agent

Le tuner et l'URL OBS n'ont jamais besoin d'être modifiés : ils lisent tout depuis trois déclarations. Ajouter une animation = créer un fichier et ajouter trois lignes.

1. Le composant — components/MonEffetBackground.js

Une fonction qui retourne { el, update, destroy }. Squelette minimal fonctionnel :

// @ts-check
/**
 * MonEffetBackground.js — description de l'effet.
 * @param {{ speed?: number, color?: string }} [options]
 * @returns {import('../types.js').ComponentInstance}
 */
export function MonEffetBackground(options = {}) {
  let speed = options.speed ?? 1;

  const canvas = document.createElement('canvas');
  canvas.style.cssText = 'position:absolute;inset:0;width:100%;height:100%;pointer-events:none;';
  const ctx = canvas.getContext('2d');

  let rafId = 0;
  function tick() {
    rafId = requestAnimationFrame(tick);
    // ... dessiner une frame ici ...
  }
  rafId = requestAnimationFrame(tick);

  return {
    el: canvas,
    update(newOptions) {
      // appliquer les nouvelles options sans recréer le canvas
      if (typeof newOptions?.speed === 'number') speed = newOptions.speed;
    },
    destroy() {
      cancelAnimationFrame(rafId); // OBLIGATOIRE — sinon l'animation continue en fantôme
    },
  };
}

Le plus simple est de partir d'une copie d'un effet existant proche (components/RainBackground.js est un bon modèle : canvas, resize, couleurs via tokens). Détail complet du contrat dans Créer un composant.

2. Le registre — deux fichiers, une ligne chacun

// component-names.js — ajouter le nom dans la liste :
'MonEffetBackground',

// component-registry.js — importer + ajouter au registry :
import { MonEffetBackground } from './components/MonEffetBackground.js';
...
MonEffetBackground,

3. Le schéma de champs — dev/component-field-schemas.js

Une entrée dans BACKGROUND_FIELD_SCHEMAS décrivant les paramètres réglables. C'est elle qui génère le formulaire du tuner :

MonEffetBackground: [
  { key: 'speed', label: 'Vitesse', type: 'number', min: 0, max: 5, step: 0.05,
    unit: '×', control: 'slider', default: 1 },
  { key: 'color', label: 'Couleur', type: 'color', default: '#C8B97A' },
],

Types disponibles : 'text', 'number', 'select' (+ choices), 'textarea', 'color' (picker + palette + CSS libre) et 'colors' (liste ajout/retrait + gradients).

Tout champ numérique doit déclarer min, max, step et control: 'slider'. Les profils communs vivent dans NUMBER_FIELD_GUIDANCE.

4. Vérifier

bun test

Les tests de cohérence existants attrapent les oublis : un nom déclaré sans factory, une factory sans schéma, etc. Ensuite, recharger le tuner — le nouvel effet est dans le menu déroulant, avec son formulaire, ses presets, et l'URL OBS le prend en charge sans rien toucher.

Performance et cycle de vie

Les canvas utilisent components/canvas-runtime.js : DPR 2 maximum en Auto, DPR 1 avec ?quality=performance. Le tuner et background.html démontent l'effet quand leur page devient invisible, puis remontent le dernier état à la reprise. Dans OBS, conserver également Arrêter la source lorsqu'elle n'est pas visible.

Récap : 1 fichier créé (le composant) + 3 lignes (nom, registry, schéma). Rien d'autre à modifier — ni background.html, ni le tuner, ni le serveur.
